O Problema

Desenvolvimento sem
estrutura custa caro

Equipes que partem direto para o código sem um processo estruturado enfrentam os mesmos padrões de falha repetidamente.

Ambiguidade de Requisitos

Decisões de design tomadas no meio da implementação geram retrabalho e débito técnico acumulado.

Scope Creep Silencioso

Sem uma especificação imutável como referência, o escopo expande a cada sprint sem que ninguém perceba.

Falta de Rastreabilidade

Impossível saber qual tarefa implementa qual user story ou qual princípio arquitetural está sendo validado.

Violações Arquiteturais

Sem princípios definidos antes do código, padrões errados se propagam por toda a base de código.

Estimativas Imprecisas

Sem tarefas geradas de contratos reais, estimativas são especulação — e bugs aparecem tarde demais.

Revisões sem Contexto

Revisores sem acesso à especificação aprovam código correto tecnicamente mas errado funcionalmente.

Etapa 1

Constitution

Estabelece os princípios arquiteturais do projeto como lei imutável. Toda feature gerada subsequentemente é validada contra esses gates antes de qualquer código ser escrito.

Os princípios definidos aqui são revisados pela equipe e aplicados automaticamente a cada ciclo do Spec-Kit.

📐 Arquitetura em Camadas controllers → services → repositories
🏗️ Bounded Contexts comunicação assíncrona entre módulos
🔌 Inversão de Dependências repositórios injetados via interfaces
🧪 Test-First Gate cobertura mínima como critério de PR
🛡️ Error Handling Global filtro único, sem try/catch em controllers
zsh — ~/projeto
~/projeto $ 
•✅ PASS  I. Layered Architecture
•✅ PASS  II. Modular Boundaries
•✅ PASS  III. Dependency Inversion
•✅ PASS  IV. Test Coverage Gate
•✅ PASS  V. Error Handling Global
›Constitution validated. Pronto para especificar features.

Etapa 5

Implement

Executa as tarefas do tasks.md em ordem, validando checkpoints entre fases e aplicando os gates da Constitution a cada módulo gerado.

Cada checkpoint garante que o código está alinhado à spec, ao plano técnico e aos princípios arquiteturais antes de avançar para a próxima fase.

✅ Phase 1 — Setup ambiente configurado
✅ Phase 2 — Foundational infraestrutura pronta
⏳ Phase 3..N — User Stories em execução sequencial
