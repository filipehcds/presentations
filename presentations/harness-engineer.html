O que é uma
LLM?

Large Language Models são redes neurais treinadas em bilhões de tokens que aprendem a representar linguagem como vetores em espaço de alta dimensão.

Predição de tokens aprende o próximo token dado um contexto
Embeddings conceitos viram vetores em alta dimensão
Attention pondera relações entre todos os tokens

Agentes de IA

O que é um
Agente de IA?

Um LLM equipado com ferramentas externas que age de forma autônoma — raciocina, executa ações e itera até atingir um objetivo.

Ferramentas expandem o LLM além do texto. Integrações (CLIs, MCPs) conectam o agente ao mundo real.
× × × ×
Integrações do Agente

CLIs · MCPs · Skills

Cada mecanismo expande o agente de forma diferente — a escolha certa impacta custo de tokens, latência e manutenção.

CLIs
bash · shell · binários
Quando usar
  • →Operações de sistema (git, docker, kubectl)
  • →Scripts e automações one-shot
  • →Sem servidor ou protocolo extra
Consumo de tokens
médio
Definição da tool ~0 tk
Comando enviado ~20–80 tk
Resultado (stdout) 100–∞ tk

Stdout vai inteiro ao contexto sem filtragem — um git log completo pode custar milhares de tokens.

$ git log --oneline -5
MCPs
Model Context Protocol servers
Quando usar
  • →APIs externas tipadas (GitHub, Slack, DB)
  • →Dados estruturados com schema definido
  • →Reutilização entre múltiplos agentes
Consumo de tokens
baixo
Schema das tools 200–800 tk
Chamada (args JSON) ~50–150 tk
Resultado (filtrado) 50–400 tk

Schema das tools é custo fixo por sessão — o resultado JSON retorna apenas os campos tipados, sem verbosidade.

⚡ mcp__github__list_prs
Skills
prompts reutilizáveis / slash commands
Quando usar
  • →Fluxos complexos multi-etapa recorrentes
  • →Comportamentos que precisam de raciocínio
  • →Encapsular lógica de negócio no prompt
Consumo de tokens
alto
Prompt da skill 500–5k tk
Ferramentas invocadas acumula por loop
Com prompt cache −90% no 2º run

Prompt da skill injeta instruções completas a cada invocação — prompt caching é essencial para amortizar o custo.

/ code-review --fix
Regra prática: prefira MCPs quando a API tem schema estável, CLIs para operações pontuais de sistema, e Skills para automatizar raciocínio recorrente — combinando os três você maximiza capacidade com o menor custo de contexto.

Etapa 1

Agentes & Tool Use

LLMs operam além do texto quando recebem ferramentas: funções que podem chamar para buscar dados, executar código ou interagir com sistemas externos.

Agentes são loops onde o modelo raciocina, age e observa resultados até atingir um objetivo — sem roteiro fixo.
Tool Calling JSON estruturado → execução real
Chain-of-Thought raciocínio explícito antes de agir
Memória scratchpad + long-term memory

Loop do Agente

1
Recebe input do usuário
Objetivo e contexto inicial carregados na janela de contexto
2
LLM raciocina e planeja
Decide qual ferramenta chamar com quais parâmetros
3
Executa ferramentas
API, DB, código Python, browser — o resultado volta ao contexto
4
Itera até concluir
Repete raciocinar → agir → observar até atingir o objetivo
5
Retorna resposta final
Resposta sintetizada entregue ao usuário com rastreabilidade

Etapa 3

RAG

Retrieval Augmented Generation — conecta o LLM a bases de conhecimento externas sem re-treinar o modelo.

O modelo responde com base em documentos recuperados em tempo real, eliminando alucinações sobre dados privados ou recentes.

Vector Database Pinecone, Weaviate, pgvector
Embedding Model text-embedding-3, voyage-3
Re-ranker filtra e reordena por relevância real
Quando usar RAG? Sempre que o conhecimento necessário muda frequentemente, é privado, ou é maior que o context window.

Pipeline RAG

1
Query do usuário
"Qual é a política de reembolso para clientes premium?"
2
Embedding da query
Transforma a pergunta em vetor numérico de alta dimensão
3
Busca vetorial (ANN)
Top-K documentos mais semanticamente similares
4
Augment + Generate
Documentos injetados no contexto → LLM gera resposta citando fontes
